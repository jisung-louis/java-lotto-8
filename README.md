## 🎯 학습 목표
- 관련된 함수를 클래스로 묶어 **객체 단위의 협력 구조**를 설계한다.
- **클래스별 단위 테스트**를 통해 동작을 검증한다.
- 2주 차 공통 피드백을 반영해 **책임 분리 / 설계 품질**을 개선한다.

---

## ✅ 과제 진행 요구 사항
- 미션은 로또 저장소를 포크하고 클론하는 것으로 시작한다.
- 기능을 구현하기 전 `README.md`에 구현할 기능 목록을 정리해 추가한다.
- Git의 커밋 단위는 README에 정리한 **기능 목록 단위**로 추가한다.
- 커밋 메시지는 **AngularJS Commit Message Conventions**을 따른다.
- 자세한 내용은 프리코스 진행 가이드 문서를 참고한다.

## 🧩 구현 기능 목록 (체크리스트)

### 1️⃣ 로또 구입 금액 처리
- [x] 구입 금액 입력받기
- [x] 입력값이 **숫자가 아닌 경우 / 1000원 단위가 아닌 경우** 예외 처리
- [x] 구입 금액에 따라 발행할 로또 개수 계산
- [x] `[ERROR]` 형식의 예외 메시지 출력 후 재입력 받기

---

### 2️⃣ 로또 발행
- [x] 로또 한 장에 6개의 중복되지 않은 번호 생성 (`Randoms.pickUniqueNumbersInRange(1, 45, 6)`)
- [x] 각 로또 번호 오름차순 정렬
- [x] 구매한 개수만큼 로또 발행
- [x] 발행 결과 출력  
  ```
  8개를 구매했습니다.
  [8, 21, 23, 41, 42, 43]
  [3, 5, 11, 16, 32, 38]
  ...
  ```

---

### 3️⃣ 당첨 번호 입력 및 검증
- [x] 쉼표(,)로 구분된 6개의 당첨 번호 입력
- [x] 숫자 범위(1~45) 검증
- [x] 중복된 숫자 존재 시 예외 발생
- [x] 보너스 번호 입력 및 검증 (중복/범위 포함)

---

### 4️⃣ 로또 결과 계산
- [x] 로또 당첨 조건 정의 (3개~6개 일치, 5개+보너스 포함)
- [x] 각 로또에 대한 일치 개수 및 보너스 여부 판단
- [x] 등수별 당첨 횟수 계산
- [x] 총 상금 및 수익률 계산 (소수점 둘째 자리 반올림)

| 등수 | 조건 | 상금 |
|------|------|------|
| 1등 | 6개 번호 일치 | 2,000,000,000원 |
| 2등 | 5개 번호 + 보너스 번호 일치 | 30,000,000원 |
| 3등 | 5개 번호 일치 | 1,500,000원 |
| 4등 | 4개 번호 일치 | 50,000원 |
| 5등 | 3개 번호 일치 | 5,000원 |

---

### 5️⃣ 출력
- [x] 구매한 로또 목록 출력
- [x] 당첨 내역 출력  
  ```
  3개 일치 (5,000원) - 1개
  4개 일치 (50,000원) - 0개
  ...
  ```
- [x] 총 수익률 출력 (예: `총 수익률은 62.5%입니다.`)
- [x] `[ERROR]` 메시지는 요구 형식 준수

---

### 6️⃣ 클래스 설계
- [ ] `Lotto` - 로또 번호 6개를 관리 및 검증
- [ ] `LottoStore` - 금액 기반 로또 발행
- [ ] `WinningNumbers` - 당첨 번호 및 보너스 번호 관리
- [ ] `LottoResult` - 등수 계산 및 통계 집계
- [ ] `InputView` / `OutputView` - 입력/출력 처리
- [ ] `Application` - 전체 실행 흐름 통합 (입력 → 처리 → 출력)

---

### 7️⃣ 예외 처리
- [x] `IllegalArgumentException` 사용 (메시지는 `[ERROR]`로 시작)
- [x] 잘못된 입력 시 해당 부분부터 다시 입력받기
- [x] `System.exit()` 사용 금지

---

### 8️⃣ 테스트
- [ ] JUnit5 + AssertJ로 도메인 단위 테스트 작성
- [ ] `Lotto`, `WinningNumbers`, `LottoResult` 클래스 테스트 필수

## 🧱 커밋 단위 가이드

| 순서 | 커밋 메시지 예시 | 설명 |
|------|------------------|------|
| 1 | `feat(domain): 로또 번호 검증 및 생성 기능 구현` | Lotto 클래스 |
| 2 | `feat(domain): 당첨 번호 및 보너스 검증 기능 구현` | WinningNumbers 클래스 |
| 3 | `feat(domain): 로또 발행 및 금액 계산 기능 구현` | LottoStore |
| 4 | `feat(domain): 당첨 결과 계산 및 수익률 로직 구현` | LottoResult |
| 5 | `feat(view): 입력 및 출력 뷰 구현` | InputView / OutputView |
| 6 | `feat(app): Application에서 전체 실행 흐름 통합` | Application |
| 7 | `test(domain): 주요 클래스 단위 테스트 추가` | JUnit5 테스트 |
| 8 | `docs(readme): 기능 목록 및 커밋 단위 정리` | README 업데이트 |
